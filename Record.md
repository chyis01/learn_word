# 背单词

**一个背单词的小软件**



流程设计

```
项目：背单词软件
重点：单词顺序策略—（转换为：给每个单词评分，根据分数判断记忆程度）
整体流程：
1.初始化：选择一本单词本-测试单词能力-根据评分确定单词的顺序
2.学习：推送单词-根据结果修改评分-存储结果-重新排序
3.时间：根据时间，遗忘程度，修改单词评分

工作内容：
1.产品的逻辑设计——策略
2.业务流程设计——出流程图
3.具体业务流设计——再说吧
4.代码实现

优化点：
1.分成学习、复习功能
2.常用单词单独保存
3.可视化小窗口
4.使用数据库
5.做成小程序（需要前端）
6.判断什么单词再也用不学了


最终：
给出一个能交互的小系统，展示推送逻辑，并且能存储结果
```



工作任务

```
一、设计
1.流程设计
2.字段设计
3.功能设计

数据处理：
1.数据获取-获取词典csv
2.数据格式处理——csv处理，选项处理，其他字段添加
3.读取数据，计算记忆等级
4.生成词典json格式文件

操作流程：
1.文件中获取词汇表，计算遗忘值，获得一个wordlist
2.根据需求，选择x个单词推送给界面
3.根据对错加减分，更新“正确”字段
4.结束时，更新wordlist，并且更新文件

学习功能：
1.按原单词表顺序给
2.只学习没有学过的
3.答案正确记忆等级+20
4.答案错误记忆等级-10
5.回答正确才不再学习

复习功能：
1.只复习学习正确的字段
2.按照记忆程度由低到高复习
3.答案正确记忆等级+20
4.答案错误记忆等级-10
5.复习正确的词当天不再复习

遗忘计算：
打开软件时，根据上次学习的日期和当天日期，适当减少记忆等级

重置历史功能：

不再学习功能：

```
